Question-2: What is the difference between innerHTML, innerText, and textContent ?

In JavaScript, "innerHTML" retrieves the complete HTML content within an element, including tags, while "innerText" only returns the visible text content of an element, ignoring HTML tags, and "textContent" retrieves all text content within an element, including hidden text like script tags, while also preserving whitespace and line breaks, unlike "innerText" which may adjust spacing based on rendering rules; essentially, "innerHTML" is for manipulating the full HTML structure, "innerText" is for getting the visible text on a page, and "textContent" is for accessing raw text content without considering styling or rendering behavior.

Question-3: What is event delegation in the DOM?

In the DOM, "event delegation" is a JavaScript design pattern where you attach a single event listener to a parent element instead of adding listeners to multiple child elements, allowing you to efficiently handle events on those child elements by leveraging the event bubbling mechanism where events propagate from the target element up the DOM tree to its parent element; this way, you can identify the specific child element that triggered the event within the parent's event handler.

Question-4: What is event bubbling in the DOM?

Bubbling in the DOM is an event propagation method. It starts with an event triggering on the innermost element and then proceeds to trigger the same event on its parent element and then its parent's parent element and so on.
